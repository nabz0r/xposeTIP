## Findings of note

**Native `title` tooltip timing is unreliable in headless capture.** The DOM-level `title="Identity glyph — generated from this target's digital fingerprint"` attribute is verified, but the actual visual tooltip on the badge required several seconds of hover in the headless Chromium session and ultimately wasn't reliably caught in screenshot 08. This is a Playwright + native browser-tooltip-delay interaction, not a product defect. A styled tooltip component (via Radix UI Tooltip or similar) would render deterministically and be screenshot-capturable; out of scope for this sprint. Logged as a tracked nice-to-have.

**S134 fast-cascade smoke could not be re-run live in this session** (Camelia's most recent scan was 48 min old, inside the 60-min skip-if-recent window mandated by the spec). The S134 stepper render is independently confirmed by the worker-log cascade timeline (4 transitions in 215 s) plus the previously-committed `s134_*.png` screenshots that captured the live render of the *same* scan ID (`7c5d2012`). If a true cold-start smoke is needed, scan a different target — but the previously-committed screenshots are direct evidence from this very scan, not a stale fixture.

**S135 fix C (post-`done` re-fetch with `setTimeout(2000)`)** is not exercised by this session because the cascade was already `done` at navigation. The code path is present and built; the previous S134 smoke that motivated this fix is the live evidence of the original bug. Future sprints can test the fix path more directly by triggering a fresh scan and monitoring the React tree as cascade transitions.

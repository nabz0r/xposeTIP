### NEW bug found during walkthrough (high impact, user-blocking)
**Consulting page is broken.** `GET /api/v1/targets?per_page=500` returns **HTTP 422 Unprocessable Entity** because the backend caps `per_page` at 100 (`api/routers/targets.py:141` — `Query(20, ge=1, le=100)`) but `dashboard/src/pages/Consulting.jsx:89` calls `getTargets('per_page=500')`. The error path renders the FastAPI validation object directly into the DOM as the string `[object Object]`, and the page falls back to "No targets in this workspace yet." — even though the workspace has 12 targets. Net effect: Consulting tier generation is unusable for any logged-in user with at least one target.

  - Suggested fix: change Consulting.jsx to `per_page=100` (or paginate), or raise the backend cap in `api/routers/targets.py` to match findings router (500). Plus fix the error renderer to stringify error objects.
  - Caveat: `dashboard/src/pages/TargetDetail.jsx:60,116` also call `getFindings(...per_page=200)`, but that's fine — findings router allows up to 500 (`api/routers/findings.py:43`).
